Deep Dive

RTOS Concurrency — How I Validate It

Using Polyspace to catch what runtime testing often misses — the non-deterministic concurrency bugs.

Task / ISR Execution Model
ISR
Highest priority · preempts all
Task_High
High priority task
Task_Low
Low priority — interruptible
Init
One-time init context
⚠ Shared variable accessed across ISR + Task_High + Task_Low
→ Race condition possible if no critical section guard
🔴
Preemption Validation Verify high-prio tasks correctly interrupt low-prio execution at shared variable access points
⚡
Race Condition Detection Polyspace flags unprotected shared variables accessed concurrently — catching bugs before hardware
🔒
Critical Section Analysis Validate that mutexes / disableInterrupts properly guard non-atomic R/W operations
📋
Task/ISR Interaction Data integrity across ISR context vs task context — non-atomic sequences under interrupt load
